Compact & Efficient Prediction

  • Retentive Network (RetNet):
    Uses a Transformer variant model architecture (~3K parameters), small enough to fit in L1 cache for fast inference.
  • CPU-Based Inference:
    Avoids slow GPU communication. Prediction runs on the application's core during idle I/O wait time, hiding latency.
  • Small Vocabulary:
    Predicting ordinals instead of full memory addresses space simplifies model complexity, boosting accuracy.

The Runtime Address Book

  • Dynamic Mapping:
    Future Maps are created at runtime, learning the memory layout as the app runs and making the system immune to ASLR.
  • Page-Specific:
    Each memory page has its own Future Map, tracking the most likely next pages to be accessed from it.
  • Swappable & Indirected:
    Future Maps can be swapped to far memory. An indirection layer (Future Map Roots) manages their locations efficiently.

Making Prefetching Timely & Effective

  • Multi-Step Lookahead:
    The model predicts several steps into the future to ensure prefetched data arrives before it's needed.
  • Asynchronous Prefetching:
    Prefetch requests are sent in the background, so the application thread isn't blocked.
  • Efficient Input Encoding:
    Uses rotary positional encoding to reuse computations for overlapping access histories, speeding up predictions.
